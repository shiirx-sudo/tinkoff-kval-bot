# portfolio-dashboard (F4.9 — read-only portfolio overview dashboard)

> 🔍 **Локальный READ-ONLY кокпит.** F4.9 рендерит **только** готовый отчёт F4.8
> (`data/reports/portfolio_dashboard_data.json`) в один HTML-кокпит через stdlib
> `http.server` (127.0.0.1). Он **не торгует**, **не ходит к брокеру**, **не читает
> токены**, **не инициализирует брокер-клиент** и **сам портфель не считает** —
> источник истины полностью в F4.8.

Цель — за 5 секунд ответить: сколько стоит портфель, сколько свободного кэша,
сколько пассивного дохода в месяц, насколько близко к цели 150 000 ₽/мес., какой
оборот к цели 60 000 000 ₽/год, настроены ли взносы, есть ли критичные риски.

## F4.9.1 — визуальный редизайн (только UI)

F4.9.1 — это **только визуальный** редизайн: модель данных F4.8, безопасность,
маршруты и поведение `/state.json` **не меняются**. Дашборд выглядит как настоящий
инвестиционный кокпит, а не как технический HTML-отчёт. Всё построено на **inline
CSS/SVG**, без JS/CDN/внешних ресурсов и без интернет-зависимостей.

Что добавлено визуально (server-rendered из F4.8 JSON):

- **Левый сайдбар-навигация** (только якорные ссылки на той же странице, без
  действий): Overview · Portfolio · Income · Turnover · Contributions · Risk ·
  Last trade · Raw JSON.
- **Шапка**: заголовок `Portfolio cockpit`, подзаголовок
  `Read-only portfolio overview from F4.8`, бейджи `READ_ONLY_SAFE`/`BLOCKED_UNSAFE`,
  `full`/`partial`, маскированный account, `generated_at`; баннер-предупреждение при
  не-localhost host.
- **KPI-карточки** с сильной иерархией и статус-цветом по смыслу (стоимость, доход/
  мес., покрытие цели, кэш, оборот YTD, PnL + взносы/безопасность).
- **Прогресс-компоненты**: бар пассивного дохода к 150 000 ₽/мес., бар оборота к
  60M/год, бар доли кэша; PnL подсвечивается красным/зелёным по знаку.
- **Аллокация — донат по весу позиций** (inline SVG из `positions[].weight_pct`,
  топ-6 + «Прочее»; **подписано `Position weight allocation`, это НЕ классы
  активов** — классы не выдумываются).
- **Доход**: столбчатый график `income_calendar_monthly` + список ближайших событий;
  при недоступном net-доходе показывается налоговое предупреждение.
- **Оборот**: прогресс к 60M, план-на-сегодня/факт/разрыв, столбики по сторонам и по
  месяцам; определение `turnover = buy+sell gross, не дивиденды/купоны` остаётся
  видимым.
- **«Что сейчас»**: исполнительные карточки (Portfolio/Income/Turnover/Contributions/
  Risk) по одному факту в каждой; явная фраза «Это диагностический дашборд, не
  торговая рекомендация».
- **Риск**: пороги отображения дашборда (топ-5 ≥ 70%, ≥ 50% позиций в минусе, кэш
  < 5%), помеченные `Display threshold, not investment advice`.
- **Позиции**: таблица перенесена ниже, читаемее — раскраска строк по знаку PnL,
  числовые колонки выровнены вправо, тикер выделен.
- **Страница «нет отчёта»** оформлена в том же каркасе.
- **Сырой JSON F4.8** — внизу, свёрнут, санитизирован.

## Что F4.9 делает и чего НЕ делает

- ✅ читает **только** локальный `data/reports/portfolio_dashboard_data.json` (F4.8);
- ✅ показывает: шапку (статус/свежесть), KPI-полосу, блок «Что сейчас», разделы
  A–G (портфель, позиции, доход/FIRE, оборот, взносы, риск, последняя сделка) и
  H — сырой JSON F4.8 (свёрнут, для отладки);
- ✅ если отчёта F4.8 нет — показывает **дружелюбную** страницу с подсказкой
  (без падения);
- ✅ маскирует account id и редактирует токеноподобные строки в state и HTML;
- ❌ **не** торгует; **нет** POST-маршрутов, **нет** `/buy` `/sell` `/cancel`
  `/retry` `/execute` `/order` и любых action-эндпоинтов; **нет** кнопок;
- ❌ **не** инициализирует брокер-клиент; **не** читает токены из окружения;
  **не** использует `TINKOFF_TOKEN` / `TINKOFF_LIVE_TRADING_TOKEN` /
  `TINKOFF_SANDBOX_TOKEN`;
- ❌ **нет** планировщика/Telegram; HTML без внешних ресурсов/CDN/JS-библиотек.

## Маршруты (только GET)

| Маршрут | Назначение |
| --- | --- |
| `GET /` | HTML-кокпит |
| `GET /state.json` | санитизированный state из F4.8 |
| прочее | `404` |
| `POST` (любой) | не поддерживается (`501`) |

## Разделы

- **Шапка**: `Portfolio cockpit`, бейдж `READ_ONLY_SAFE`/`BLOCKED_UNSAFE`, бейдж
  свежести `full`/`partial`, `generated_at`, маскированный account.
- **KPI-полоса**: стоимость портфеля, кэш и %, пассивный доход/мес., покрытие цели,
  оборот YTD vs 60M, PnL портфеля, статус взносов, безопасность.
- **«Что сейчас»**: факты простым языком (без советов), включая покрытие цели
  150 000 ₽/мес., прогресс оборота и требуемый дневной оборот, состояние взносов,
  риск-сводку, и явную пометку «диагностический дашборд, не торговая рекомендация».
- **A** Обзор портфеля · **B** Таблица позиций · **C** Пассивный доход/FIRE ·
  **D** Оборот (60M/год; buy+sell gross, не дивиденды/купоны) · **E** Взносы ·
  **F** Риск/концентрация · **G** Последняя сделка (вторично) · **H** Сырой JSON.

В разделе риска показываются **пороги отображения дашборда** (не инвестиционный
совет): топ-5 ≥ 70%, доля убыточных позиций ≥ 50%, кэш < 5%.

## Запуск / остановка

```powershell
# 1) Сначала сформируйте/обновите данные F4.8 (read-only):
python main.py portfolio-dashboard-data --live-account-id <ACCOUNT_ID>

# 2) Затем запустите кокпит и откройте в браузере http://127.0.0.1:8766
python main.py portfolio-dashboard --host 127.0.0.1 --port 8766
# остановить: Ctrl+C
```

| Опция | По умолчанию | Назначение |
| --- | --- | --- |
| `--host` | `127.0.0.1` | адрес привязки (другой host → предупреждение) |
| `--port` | `8766` | порт |
| `--report-path` | `data/reports/portfolio_dashboard_data.json` | отчёт F4.8 (чтение) |

Это отдельная команда от F4.7 `dashboard` (технического просмотрщика) — F4.7 не
ломается.

## Настройка взносов

Скопируйте `config/contribution_plan.example.json` в
`data/config/contribution_plan.json` и заполните (локальный файл, не коммитится).
Без него F4.8 ставит `contributions_tracking_enabled=false`, и F4.9 показывает
подсказку в разделе «Взносы».

## Безопасность

- Слушает только `127.0.0.1` по умолчанию; другой `--host` → предупреждение в
  консоли и на странице.
- Значения токенов никогда не читаются/не печатаются/не попадают в HTML; account id
  маскируется; токеноподобные строки редактируются рекурсивно.
- Только чтение: не пишет `.env`, `data/config/*`, не мутирует портфель/config.

## Дальше

F4.9 — только просмотр. Чтобы обновить цифры, перезапустите F4.8 и обновите
страницу. Торговля/действия в дашборд не добавляются.
